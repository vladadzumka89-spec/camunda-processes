# Ролі виконавців Camunda

## Де знайти ролі

**Odoo → Проєкт → Налаштування → Ролі процесів Camunda**

Модель: `x_camunda`

---

## Поля ролі

| Поле | Що заповнити |
|------|-------------|
| **Назва** | Назва ролі українською (напр. "Технічна підтримка") |
| **Код ролі** | Латиницею, snake_case (напр. `tech_support`) |
| **Вид процесу Camunda** | Обрати для якого процесу ця роль |
| **Підрозділ** | Обрати підрозділ (якщо роль прив'язана) |
| **Виконавці** | Обрати одного або кількох користувачів |

---

## Існуючі ролі

| Назва | Код |
|-------|-----|
| Ревізор комерційний | `revisor_commercial` |
| Фінансист | `financist` |
| Бухгалтер | `accountant` |
| Діловод | `dilovod` |
| Технічна підтримка | `tech_support` |
| Аналітик | `analyst` |
| Рекрутер | `recruiter` |

---

## Як прописати в BPMN

В body User Task додай поле:

```
x_studio_camunda_role_code: "tech_support"
```

Автоматизація "Camunda: призначення виконавця" знайде роль по коду і підставить виконавців в задачу.

### Інші способи призначення виконавця:

По керівнику підрозділу:
```
x_studio_camunda_user_id: x_studio_camunda_manager_user_id
```

По конкретному ID користувача:
```
x_studio_camunda_user_id: 6
```

---

## Важливо

- Поле `x_studio_camunda_role_code` (char) має існувати в моделі `project.task`
- Webhook URL має бути **прод** (`https://o.tut.ua/web/hook/...`), не демо
- Якщо в ролі кілька виконавців — задача призначається всім, перший хто виконає — решта знімаються
