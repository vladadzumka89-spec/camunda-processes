# Налаштування Odoo для нового процесу Camunda

Проведи користувача покроково. На кожному кроці чекай підтвердження.

## Крок 1: Створити проєкт в Odoo

Скажи користувачу:

> Для кожного бізнес-процесу в Odoo має бути **окремий проєкт**.
>
> 1. Перейдіть в **Проєкт** → **Новий**
> 2. Введіть назву проєкту (наприклад "Оплата рахунків")
> 3. Збережіть
> 4. Зверніть увагу на **ID проєкту** в URL (наприклад `/web#id=252` → ID = `252`)
>
> Цей ID знадобиться для налаштування BPMN-процесу.

Запитай:
> Який **ID проєкту** у вас? (число з URL)

Запамʼятай цей ID — він буде використовуватись як `_id` в BPMN.

---

## Крок 2: Створити "Вид процесу Camunda" в Odoo

Скажи користувачу:

> Потрібно створити запис для нового процесу в Odoo.
>
> 1. Перейдіть за URL: `odoo.dev.dobrom.com:2689/odoo/x-camunda-process-type` (для демо) або `o.tut.ua/odoo/x-camunda-process-type` (для прод)
> 2. Натисніть **Новий**
> 3. Заповніть:
>    - **Name** — назва процесу українською (наприклад "Оплата рахунків")
>    - **processDefinitionId** — Process ID з BPMN-схеми (наприклад `oplata-rakhunkiv`)
> 4. Збережіть

Запитай:
> Який **Process ID** у вашому BPMN? (латиницею, наприклад `oplata-rakhunkiv`)
> Яка **назва процесу** українською?

Після відповіді покажи що саме вписати в кожне поле.

**Нагадування:** Process ID в Odoo ПОВИНЕН точно збігатися з ID процесу в BPMN-схемі. Якщо процес в пулі — Process ID знаходиться при кліку на рамку пулу → права панель → **Process ID**.

## Крок 3: Створити поля в Odoo

Скажи користувачу:

> Тепер потрібно створити поля для процесу в Odoo.
>
> **ВАЖЛИВО:** Всі поля ОБОВ'ЯЗКОВО мають починатися з `x_studio_camunda_`
> Без цього префіксу воркер ігнорує поле і дані не передаються в Camunda.
>
> Перейдіть в модель завдання (project.task) через Odoo Studio:
> 1. Відкрийте будь-яке завдання
> 2. Увімкніть **Studio** (іконка палітри вгорі)
> 3. Додайте потрібні поля

Запитай:
> Які дані потрібно передавати між Odoo та Camunda в цьому процесі?
> (наприклад: сума рахунку, номер договору, відповідальний, дата тощо)

Після відповіді — запропонуй список полів з правильними назвами. Формат:

| Поле | Технічна назва | Тип |
|------|---------------|-----|
| Сума рахунку | `x_studio_camunda_invoice_amount` | float |
| Номер договору | `x_studio_camunda_contract_number` | char |
| Відповідальний | `x_studio_camunda_responsible_user_id` | many2one (res.users) |

**Правила іменування:**
- Префікс завжди `x_studio_camunda_`
- Далі — назва англійською в snake_case
- Тип поля залежить від даних (char для тексту, float для сум, boolean для так/ні, many2one для зв'язків)

## Крок 4: Налаштувати видимість полів

Скажи користувачу:

> Створені поля мають бути **видимі тільки для свого проєкту**, а не для всіх завдань.
>
> Для кожного створеного поля `x_studio_camunda_*`:
> 1. В Odoo Studio клікніть на поле
> 2. Зліва знайдіть **Прихований** → натисніть **Умовно**
> 3. Додайте умову: **Проєкт** ≠ *ваш проєкт* (наприклад "Оплата рахунків")
>    - Це означає: поле приховане для всіх проєктів, крім вашого
> 4. Повторіть для кожного поля
>
> Так поля не будуть захаращувати форму завдань в інших проєктах.

---

## Крок 5: Привʼязати вид процесу до завдання

Скажи користувачу:

> Тепер потрібно вказати вид процесу в завданні:
> 1. Відкрийте завдання в Odoo
> 2. Знайдіть поле **Вид процесу Camunda**
> 3. Оберіть створений вид процесу (наприклад "Оплата рахунків")
> 4. Збережіть

## Крок 6: Перевірка

Скажи користувачу:

> Перевіримо що все налаштовано:
> 1. Перемістіть завдання на стадію **"В процесі"**
> 2. Подивіться в chatter завдання — має з'явитись повідомлення **"Camunda Process Started"**
> 3. Поле **"Ключ процесу Camunda"** має заповнитись числом
>
> Якщо замість цього з'явилась помилка — покажіть мені скрін.

### Типові помилки на цьому кроці:

| Помилка | Причина | Рішення |
|---------|---------|---------|
| Нічого не відбувається | Автоматизація не налаштована або не тригериться | Перевірити автоматизацію "Запуск процесу в Камунда" |
| `processDefinitionId: EMPTY` | Вид процесу обрано, але `processDefinitionId` порожній | Відкрити запис в `x_camunda_process_type` → заповнити поле |
| `404 NOT_FOUND` | Process ID не збігається з BPMN або BPMN не задеплоєний | Перевірити що ID однаковий в Odoo і в BPMN, задеплоїти BPMN |
| `Camunda OAuth2: відсутня конфігурація` | Не налаштовані системні параметри Camunda | Звернутись до адміністратора |
